# TelopGuard: AI Telop Correction Tool

TelopGuard（テロップガード）は、動画編集者向けのテロップ校正ツールです。
Google Gemini APIを使用して、テロップの誤字脱字、表記揺れ、ニュアンスの違和感を自動検出し、修正案を提示します。

## 主な機能
- **自動校正**: テキストファイル（.txt）をドラッグ＆ドロップするだけでAIがチェック。
- **ハイブリッド判定**:
  - **①機械的チェック**: 数字の半角統一、不要な行の検出など、ルールベースのチェックです。
  - **②AIによる意味チェック**: 文脈に応じた誤変換（例：上手くいく vs 上手く言う）やニュアンスをチェックする自然言語的チェックです。
- **重複行アラート**: 前の行と同じ内容が含まれている場合、即座に警告。
- **BYOK (Bring Your Own Key)**: APIキーはブラウザに保存され、サーバーには送信されません。

## 使い方
1. **APIキーの取得**:
   - [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスし、APIキー（無料枠あり）を取得してください。
2. **アプリの起動**:
   - 右上の「API KEY」欄にキーを入力し、必要なら「Test」で接続確認をします。
3. **チェック実行**:
   - VrewやPremiere Proから書き出したテキストファイル（.txt）をアップロード。
   - 自動的に ①機械的チェック が行われます。
   - 「Run AI Check」ボタンを押すと、②AIによる意味チェックが実行されます。

## 配布・デプロイ方法（他の人と共有する場合）
このアプリは静的なWebサイトとして動作します。以下のいずれかの方法で共有できます。

### A. Vercelで公開（推奨）
最も簡単に、URLで共有できる方法です。
1. [Vercel](https://vercel.com/)のアカウントを作成。
2. このプロジェクトフォルダをGitHubにアップロード、またはVercel CLIを使ってデプロイ。
3. 発行されたURLを共有するだけで、誰でも自分のAPIキーを入れて使えます。

### B. ローカルで実行
エンジニア以外の方には推奨しませんが、`.zip`等で渡す場合：
1. `Node.js` が必要です。
2. フォルダ内で `npm install` -> `npm run dev` を実行するとブラウザで開けます。

## 最近のアップデート (v3.0.0)
今回のアップデートで、動画チャンネルごとのルールの切り替え機能追加など、大幅な機能強化を行いました！

1. **チャンネル専用プロファイルの切り替え機能 (Multi-Profile)**
   - 画面左上のドロップダウンから担当チャンネル（例：`nakano-shiki`、`skill-kakutoku` 等）を選択できるようになりました。選んだチャンネルに紐づく専用のトンマナや表記揺れ辞書が即座に反映されます。
   - `npm run dev:nakano-shiki` などのコマンドで、特定チャンネル専用モードとしてアプリを立ち上げることも可能です。
   - 各チャンネルのローカル辞書（`src/dictionaries/*.ts`）の記述ブロックをA〜Dに整理し、新しいチャンネルの追加やルール保守が容易になりました。

2. **操作性の向上 (UI/UX)**
   - 読み込んだファイルを削除する「クリア」ボタンを、右側のアクションボタン群から、左側の「テロップ原稿」ファイル名のすぐ右隣へ移動。操作の文脈に沿った配置に改善しました。
   - テロップ原文および修正テキスト部分のフォント表示バグ（一部が明朝体で表示されてしまう問題）を解消し、綺麗なゴシック体で統一表示されるように修正しました。
   - ショートカットキー設定画面（⚙️設定）に「初期設定に戻す」ボタンを追加し、操作ミスなどでキーが上書きされてもすぐにデフォルト状態にリセットできるようになりました。

3. **Gemini API 接続テストとモデル選定の安定化**
   - Gemini APIの最新仕様（モデル名の末尾に`-001`などのバージョン番号が付くケース）に対応。「接続テスト」実行時に最新モデルが正しくリストアップされるようアルゴリズムを刷新しました。
   - 画像解析やツール連携用の特殊モデル（`customtools` や `image` 等）を自動で除外するようにし、テキスト校正として純粋に機能する言語モデルだけが優先されるよう最適化しました。

4. **より賢くなった機械的ルール（誤検知修正）**
   - 頻出する「とき／時」のひらがな統一チェックにおいて、「ごとき」や「〜しておき」などの言葉で発生していた余計な誤判定（False Positive）を、高度な正規表現（否定の先読み・戻り読み）を用いてきれいに抑制しました。
