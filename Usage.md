# 🛡️ TelopGuard v2.4 使い方マニュアル

TelopGuardは、Premiere ProやVrewから書き出したテロップのテキスト形式ファイルを読み込み、「1. 機械的な一括修正」と「2. AIによる文脈や誤字のチェック」を連続で行う校正ツールです。

---

## 🚀 基本的な使い方

### 1. APIキーの設定（初回のみ）
TelopGuardの「AIチェック機能」を利用するには、GeminiのAPIキーが必要です。
1. お使いのGoogleアカウントで [Google AI Studio](https://aistudio.google.com/app/apikey) にアクセスし、APIキー（`AIza` から始まる文字列）を取得します。
2. TelopGuardの画面右上にある「**API KEY**」欄に貼り付けます。
3. すぐ横の「**接続テスト**」ボタンを押して `✅ 接続成功！` と出れば準備OKです。（一度設定すればブラウザの「Local Storage（ローカルストレージ）」という機能を使ってお使いのパソコン内に自動保存されます）
> 💡 **【注意】** Cookieなどの「サイトデータの保存」をブロックする設定にしているブラウザや、シークレットモード（プライベートブラウズ）で開いている場合は、画面を閉じるたびにAPIキーが消えてしまいますのでご注意ください。

#### 🔑 APIキーの取得手順（所要時間：約2分）
Googleアカウントがあれば、誰でも無料で利用枠付きのAPIキーを取得できます。

1. **Google AI Studioにアクセスする**
   [Google AI Studio - Get API Key](https://aistudio.google.com/app/apikey) を開き、Googleアカウントでログインします。
2. **APIキーを作成する画面を開く**
   画面内右上にある **「APIキーを作成（Create API key）」** ボタンをクリックします。
3. **キー名とプロジェクトを設定する**
   「新しいキーを作成する」という画面が表示されたら、以下のように設定します。
   - **キー名の設定：** そのままでOKです（例：`Gemini API Key` 等、お好きな名前で構いません）。
   - **インポートしたプロジェクトを選択：** プルダウンを開き、「**＋ プロジェクトを作成**」をクリックします。（※他の何かのプロジェクトが既にプルダウンにあっても、混同や無料枠の共有化を避けるため、毎回新しく作成するのが一番確実です）
4. **生成されたキーをコピーする**
   画面に `AIza` から始まる長い文字列が表示されたら、コピーボタン（📋マーク）を押してください。

#### 🆘 エラーが出た場合の対処法
> [!WARNING]
> **「サービスが無効(SERVICE_DISABLED)」といったエラーが表示される場合**
> ごくまれに手動で作成したAPIキーで発生します。その場合、上記手順に戻って「Google AI Studio」から新しく作り直すのが一番確実で簡単です。

#### ⏱️ 無料枠の上限（Quota exceeded）に達した場合
TelopGuardはAIの見落としを防ぐため、「テキストを約30行ずつに小分けにして、裏側でAIに何十回も連続チェックを依頼」しています。そのため無料枠の利用制限（1分間あたりの通信制限など）に引っかかることがあります。

**対処法：**
1. **翌日まで待つ：** 制限は毎日、日本時間の16〜17時頃にリセットされます。
2. **別のGoogleアカウントで再発行する（すぐ使いたい場合）：** 無料枠は「Googleアカウント（プロジェクト）」単位で管理されています（同じアカウントでキーを複数作っても枠は増えません）。仕事用や別管理用の**別のGoogleアカウント**にログインし直して、新しいAPIキーを発行してください。

### 2. ファイルの読み込み
Premiere Pro や Vrew から書き出したテロップファイル（`.txt`）を、画面左側の「ドラッグ＆ドロップ」エリアに直接放り込むか、クリックして選択します。

### 3. Step 1: 機械的チェック（自動実行）
ファイルを読み込んだ瞬間に、以下のルールベースの修正が**自動で適用**され、左右に並んで表示されます。
*   全角・半角の統一（英単語や記号、数字を正しい幅に）
*   無駄な空白の削除
*   「えー」「あのー」などの不要語（フィラー）のハイライト化
*   URLや英語のみの行の除外（AIチェックから除外するための処理）

この段階ではAPIは使われないため無料で一瞬で終わります。左カラムが「元テキスト」、右カラムが「分析結果」です。**`✔ OK` 以外の箇所**を確認し、問題なければ次のAIチェックに進みます。

### 4. Step 2: AI 意味・文脈チェック
画面右上の **「▶ AIチェック実行」** ボタンを押します。
AIが数十行ずつ区切ってテキストを読み込み、以下の分析を行います。（※この時点でAPI의利用枠が少し消費されます）

*   **同音異義語の誤変換**（例：上手く「言う」⇔「行く」）
*   前後の文脈から判断した不自然な誤字脱字
*   過去のプロジェクト固有の名称・用語（中野式ルール等）との照合

AIの処理は、途中で制限（Quota）に引っかかりそうになると、自動的に3秒ほど待機したり、他のAIモデル（ProやFlash等）へ裏側で切り替えながら最後まで完走するよう設計されています。エラーが出てもしばらく待ってみてください。

### 5. 結果の確認とChatworkへの報告
処理が終わると、修正すべき箇所が **Critical（赤） / Warning（黄） / Info（青）** のレベル別にハイライトされて表示されます。

*   **修正案を採用する場合**:
    右端の `📋 Copy` ボタンを押すと、修正案のテキストだけがコピーされるので、ご自身のPremiere Pro等に貼り付けて修正してください。
*   **レポートを提出・報告する場合**:
    画面右上の **「📑 レポートをコピー」** ボタンを押すと、AIが指摘した箇所の「原文」と「修正案」が誰が見ても分かるレポート形式で丸ごとクリップボードにコピーされます。そのままChatworkに貼り付けて報告に使用できます！
    ※ AIがWeb検索で確認した「実在の固有名詞」等は、レポートの最後に「次回からの辞書登録候補」としても出力されます。

## 💡 知っておくと便利な機能

*   **🔄 再読み込みボタン**
    テキストデータを読み込んだ後、「あ、元のテキストファイルの中身ちょっと間違ってたから修正して保存し直した！」という時に押すと、ブラウザをリロードせずに最新のファイルを瞬殺で読み込み・再チェックしてくれます。
*   **クリア機能**
    現在読み込んでいるテキストデータを単に破棄（リセット）したい時に押してください。
